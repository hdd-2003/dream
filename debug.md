## [OPEN] 梦境星河：加载更慢、背景图未生成

### 症状
- 仪式页（“凝聚星辰中...”）停留时间很久，体验差
- 结果页背景图仍未出现（期望：根据梦境内容生成的图片作为背景）

### 期望
- 输入梦境后，尽快完成解读并进入结果页
- 结果页背景图稳定生成/显示，并且失败时有明确降级与提示

### 环境
- OS: macOS
- 前端：Vite + React + TS

### 复现步骤（用户侧）
1) 打开首页 → 记录梦境 → 输入任意梦境文本
2) 点击“凝聚星辰”
3) 观察：仪式页停留时长、是否进入结果页、背景图是否出现

### 假设（待验证）
1) 解读接口请求耗时过长或卡住（缺少超时/重试/并发阻塞）
2) 背景图生成接口耗时过长/失败（请求被拦截、返回非图片、或 4xx/5xx）
3) 图片实际请求成功但显示逻辑被隐藏（onError 触发/样式覆盖/被置为 display:none）
4) 图片 URL 过长或包含特殊字符导致生成端拒绝/浏览器请求失败
5) 某些环境下跨域/CSP/HTTPS mixed content 导致图片加载失败

### 证据采集计划
- 在「提交→解读请求→解析→生成图片 URL→图片 load/error」关键点上报结构化日志
- 记录耗时（ms）、HTTP 状态码、返回体关键字段、图片 onload/onerror
